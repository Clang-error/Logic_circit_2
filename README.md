해밍코드 출력 프로그램

설계서
1. main 함수는 단순 함수 호출역할을 담당
2. bit_Size 함수에서 비트사이즈 입력을 담당
3. need_Parity 함수로 필요한 해밍코드에서의 패리티비트갯수 연산담당
입력받은 데이터값을 활용. 2^p >=(d+p+1) (여기서 p는 패리티비트, d는 데이터사이즈.)
4. input_Data 함수에서 비트사이즈만큼 세부 데이터(0or1)을 받아 저장함
검증로직 추가, 배열사이즈만큼 데이터를 입력받지 않았다면 루프문으로 다시 입력받기,
입력받은데이터가 0 또는 1이 아니라면 다시 입력받기
5. set_Hamming 함수에서 각 맞는 위치 2^n 위치에 패리티비트위치를 배열에 저장 후 
P1 = D3 + D5 + D7 + D9 ... = (1의 자릿수에 X인 모든 데이터비트)
P2 = D3 + D6 + D7 + D10 + D11 ... =  (10의 자릿수에 있는 1X인 모든 데이터비트)
P4 = D5 + D6 + D7 + D12 ... = (100의자리에 있는 1XX인 모든 데이터비트)
P8 = D9 + D10 + D11 + D12 ... =(1000자리에 있는 1XXX인 모든 데이터비트)
   (여기서 +는 XOR연산자)
input_Data로 받은 세부 데이터값을 배열에 차례대로 저장
6. PrintHammingCode 함수로 전체 해밍코드를 출력

설계된 파일
1.Bit_Size.c
2.Need_Parity.c
3.Input_Data.c
4.Set_Hamming.c
5.Print_Hamming_Code.c
6.main.c

5.PrintHammingCode //Print_Hamming_Code.c

메모
1. need_hamming 값을 리턴받을것인가 전역으로 쓸 것인가 ?
2. 리턴형 함수는 어디에서 사용할 것인가 ? 
3. 패리티비트의 개수를구하고, 데이터비트를 어떻게 합쳐서 배열에 넣을 것인가 ?
4. 패리티비트의 저장은 일차원인가 이차원인가? 2차원으로 쓴다면 저장자체는 간편해짐.

참고사항
1. 입력받는 데이터의 개수가 제공됨.
2. 입력받는 데이터는 임의의 비트가 제공됨.
3. 입력받은 데이터로 해밍코드를 작성하는 프로그램 설계
4. 패러티비트는 짝수형
5. 해밍코드를 만들 때 패러티비트 수 연산해서 계산
6. 2^p ≥ d + p + 1 (패러티비트 수 계산 공식 이떄 d는 데이터의 개수)
7. 4.8 해밍코드 완성프로그램 추가설명
8. 설계서 조원 논의후 '오늘까지' 제출하기
9. 설계서는 sds@tu.ac.kr로 전송
10. 송신처리 - 수신처리
11. 임의의 입력비트가 만약 8비트다. <-- 데이터비트가 8비트.
12. 해밍코드에서 패리티비트의 개수 (입력받은 데이터값은 패리티비트가 아님.)
13. 비트사이즈 받고, 패리티비트를 구한다음에 비트값을 받는거다.
14. 패리티비트값을 결정.
15. 에러검출기 기능까지는 필요없음.
