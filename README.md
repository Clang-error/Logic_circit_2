해밍코드 출력 프로그램

설계서
1. main 함수는 단순 함수 호출역할을 담당
2. control에서 전체 함수들 흐름제어역할을 담당
3. bit_Size 함수에서 비트사이즈 입력을 담당
4. need_Parity 함수로 필요한 해밍코드에서의 패리티비트갯수 연산담당
입력받은 데이터값을 활용. 2^p >=(d+p+1) (여기서 p는 패리티비트, d는 데이터사이즈.)
5. input_Data 함수에서 비트사이즈만큼 세부 데이터(0or1)을 받아 저장함
검증로직 추가, 배열사이즈만큼 데이터를 입력받지 않았다면 루프문으로 다시 입력받기,
입력받은데이터가 0 또는 1이 아니라면 다시 입력받기
6. set_Parity 함수에서 각 맞는 위치 2^n 위치에 패리티비트위치를 배열에 저장 후 
P1 = D3 + D5 + D7 + D9 ... = (1의 자릿수에 X인 모든 데이터비트)
P2 = D3 + D6 + D7 + D10 + D11 ... =  (10의 자릿수에 있는 1X인 모든 데이터비트)
P4 = D5 + D6 + D7 + D12 ... = (100의자리에 있는 1XX인 모든 데이터비트)
P8 = D9 + D10 + D11 + D12 ... =(1000자리에 있는 1XXX인 모든 데이터비트)
   (여기서 +는 XOR연산자)
연산 후 저장값은 1차원 배열사용해도 충분.
input_Data로 받은 세부 데이터값을 배열에 차례대로 저장
7. PrintHammingCode 함수로 전체 해밍코드를 출력
입출력예시 10111011(입력) -> 패러티개수연산-> 맞는 위치에 패러티비트를 넣음->001101111011(출력) (데이터비트 8개 패러티비트4개)

프로그램흐름도

main (단순호출)-> control(전체 흐름제어 함수)->bit_Size(비트사이즈 입력) -> 
need_Parity(입력받은 비트사이즈를 기준으로 패러티비트 개수 연산) -> 
input_Data(비트사이즈만큼 데이터를 0또는 1로 입력받음) -> set_Hamming(알맞은 위치에 패러티비트 저장) -> 
print_Hamming_Code(한 배열에 저장된 패러티비트와 데이터비트 전체출력하는 함수)

참고사항
1. 입력받는 데이터의 개수가 제공됨.
2. 입력받는 데이터는 임의의 비트가 제공됨.
3. 입력받은 데이터로 해밍코드를 작성하는 프로그램 설계
4. 패러티비트는 짝수형
5. 해밍코드를 만들 때 패러티비트 수 연산해서 계산
6. 2^p ≥ d + p + 1 (패러티비트 수 계산 공식 이떄 d는 데이터의 개수)
7. 4.8 해밍코드 완성프로그램 추가설명

9. 설계서는 sds@tu.ac.kr로 전송
10. 송신처리 - 수신처리
11. 임의의 입력비트가 만약 8비트다. <-- 데이터비트가 8비트.
12. 해밍코드에서 패리티비트의 개수 (입력받은 데이터값은 패리티비트가 아님.)
13. 비트사이즈 받고, 패리티비트를 구한다음에 비트값을 받는거다.

설계이유

이 프로그램은 입력된 데이터 비트로부터 해밍코드를 생성하는 기능을 수행합니다
해밍코드는 데이터비트와 패리티비트의 위치가 정해져있고, 패리티 비트의 개수도 입력 데이터크기에 따라
달라지므로, 기능을 한 번에 처리하기보다 역할별로 함수를 분리하는 방식으로 접근하는 것이 가장 
적합하다고 생각했습니다.
이와 같이 입력, 계산, 저장, 출력의 과정을 각각 함수로 나누어 설계함으로써 프로그램의 가독성과 유지보수성을 높이고, 
각 기능을 독립적으로 관리할 수 있도록 하였습니다.
